I have just downloaded version 1.1 of netscape for Solaris 2.3
(SPARC). I had hoped that it would fix some problems that we
experienced with version 1.0, but it does not.

PROBLEM 1: We use transparent, interlaced GIF images. The Macintosh
version of netscape and a Unix version of Mosaic, along with one or two
other web browsers, have no problem with the images. However, the
Solaris version of netscape, including version 1.1, always garbles
them, although sometimes in different ways. For example, an image
might appear one time as a broken icon and another time as a black or
technicolor rectangle of the correct dimensions. The randomness might
be related to the order or size of our window history stack.
Interestingly, we rarely see this problem with web pages retrieved from
external sites, but it happens constantly with pages served on our
internal ethernet. That causes me to wonder if the problem is related
to timing and network speed.

I have attached, in uuencode format, a compressed Sun raster screen
dump of our home page. You can access the page over the Internet to
see if you witness the same problem. I'm guessing that you will not.

PROBLEM 2: This may be related to problem 1. A similar problem occurs
with even simple images, such as X bitmap icons. I've attached another
image exhibiting that problem. The blobs to the left of the file names
are the normal icons used for directory listings. Again, this problem
occurs only with netscape on Solaris; we have not seen it with other
browsers or on other platforms.

PROBLEM 3: Another problem apparently unique to netscape on Unix (or
at least on Solaris 2) involves incomplete input. I will often open a
URL known to have valid content only to have netscape display only part
or none of it. This might also be related to some type of network
problem, although I believe I have seen this particular problem even with
external URLs.

Here's an example. I have an HTML file with the following content:

<html>
<head>
<title>Employee Home Pages</title>
</head>
<body>
<h2>Employee Home Pages</h2>
<ul>
<li><a href="‾tro/">Tom Olin</a>
</ul>
</body>
</html>

With Mosaic, it displays as expected. With Solaris 2 netscape, the
page displays nothing, and View Source reveals the following content:

<html>
<head>
<title>Employee Home Pages</title>
</head>
<body>


That's all we get.
